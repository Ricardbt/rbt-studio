How I work
Step 01
Understand the friction

Before interface, before architecture — what is the actual moment of difficulty for the person using this? Most product problems are misdiagnosed at the brief stage.

Step 02
Design the interaction

How should it feel to use? What are the states, the transitions, the moments of feedback? The interface is designed before a line of code is written.

Step 03
Build with intent

Full-stack implementation — frontend architecture, backend logic, AI integration — with the experience held constant throughout. No handoff gaps.

Step 04
Leave it ownable

Systems documented, components composable, logic legible. What I build should be extendable by others — not dependent on my continued involvement.
